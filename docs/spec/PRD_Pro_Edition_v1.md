## 1. 产品定位

### 1.1 一句话定位

**PromiseLink 专业版 — 让非技术用户零配置享受AI关系经营能力，随身携带的关系助手。**

### 1.2 核心价值主张

| 维度 | 基础版 (Basic) | 专业版 (Pro) |
|------|---------------|--------------|
| **目标用户** | 技术用户（开发者/极客） | 非技术用户（商务人士如许总） |
| **核心痛点** | 愿意自配API Key，追求隐私 | 不懂技术，要"开箱即用" |
| **定价** | 免费 (AGPL v3 开源) | ¥29/月(早鸟) / ¥49/月(常规) |
| **AI能力** | 用户自备LLM API Key | 云端AI网关提供（用户无需配置） |
| **数据位置** | 用户本地（数据从不出家门） | 用户本地（网关仅中继，不存业务数据） |
| **前端入口** | Taro H5（局域网浏览器） | 微信小程序（随时随地） |
| **输入方式** | 纯文本（手动输入+粘贴） | 文本+语音+名片扫描+邮件+微信转发+CSV |

### 1.3 产品定位三要素

1. **零配置AI** — 用户不需要申请/配置任何API Key，网关统一提供DeepSeek/Moka AI能力。非技术用户激活专业版后即可使用全部AI功能，无需理解API Key、Base URL、模型名等技术概念。

2. **多模态输入** — 语音录入、名片扫描、邮件同步、微信转发、CSV批量导入。覆盖商务人士的所有信息来源场景，从"手动打字记录"升级为"说话/拍照/转发即记录"。

3. **随时随地访问** — 微信小程序为主入口，不受局域网限制。用户在开车、出差、会议间隙都能通过手机访问关系数据，真正实现"随身携带"。

### 1.4 与基础版的关系

**同代码库 + 配置区分**（详见§7）:
- 基础版和专业版共用同一套Python代码库
- 通过 `APP_EDITION` 环境变量区分（`basic` / `pro`）
- Pro-only路由在应用启动时条件注册，basic模式下路由不存在（返回404）
- 基础版开源（AGPL v3），专业版网关闭源（Open Core模型）

## 7. 与基础版的关系

### 7.1 同代码库 + 配置区分

**核心设计**: 基础版和专业版共用同一套Python代码库，通过配置区分，而非分支维护。

```python
# config.py
app_edition: str = "basic"  # "basic" or "pro"
ai_mode: str = "local"      # "local" or "relay"
```

### 7.2 版本切换机制

| 机制 | 基础版 | 专业版 | 实现 |
|------|--------|--------|------|
| `APP_EDITION` | `basic` | `pro` | `config.py` field_validator |
| `AI_MODE` | `local` | `relay` | `config.py` field_validator |
| 路由注册 | 基础路由 | 基础+Pro路由 | `main.py` 条件注册 |
| `LLM_API_KEY` | 用户自填 | 无（网关代理） | 环境变量 |
| `RELAY_GATEWAY_URL` | 无 | `wss://gw.promiselink.ai/relay` | 环境变量 |

### 7.3 功能矩阵

| 功能 | 基础版 | 专业版 | 区分方式 |
|------|:---:|:---:|----------|
| 事件录入（手动/文件上传） | ✅ | ✅ | 共用 |
| 实体抽取与归一 | ✅ | ✅ | 共用 |
| Todo生成与状态机 | ✅ | ✅ | 共用 |
| 承诺双向追踪 | ✅ | ✅ | 共用 |
| 仪表盘 | ✅ | ✅ | 共用 |
| 关系简报 | ✅ | ✅ | 共用 |
| 语音助手（ASR/NLU） | ❌ | ✅ | Pro路由条件注册 |
| 语音查询 | ❌ | ✅ | Pro路由条件注册 |
| 媒体处理（TTS/OCR） | ❌ | ✅ | Pro路由条件注册 |
| 邮件同步 | ❌ | ✅ | Pro路由条件注册 |
| 微信转发 | ❌ | ✅ | Pro路由条件注册 |
| CSV批量导入 | ❌ | ✅ | Pro路由条件注册 |
| 隐私数据管理 | ❌ | ✅ | Pro路由条件注册 |
| 网关中继 | ❌ | ✅ | `RELAY_GATEWAY_URL`设置即启用 |
| AI网关代理 | ❌ | ✅ | `AI_MODE=relay`时启用 |

### 7.4 许可证模型（Open Core）

| 组件 | 许可证 | 合规策略 |
|------|--------|----------|
| 基础版代码 | AGPL v3 | 开源，任何人可使用/修改/分发 |
| 专业版网关代码 | 闭源 | 网关是独立服务，不修改基础版代码，不触发AGPL传染 |
| relay_client模块 | AGPL v3 | 随基础版开源，用户自行部署 |
| 前端小程序 | 闭源 | 独立前端，不链接基础版代码 |

**Open Core模型**: 基础版开源是隐私的技术保证（用户可审计代码确认数据不出本地），专业版闭源的是网关服务（商业价值在AI代理+计费，不在代码）。

### 7.5 升级路径

```
基础版用户 → 付费激活专业版 → 输入PRO_LICENSE_KEY → 网关验证 → 签发JWT
                                    ↓
                            设置 RELAY_GATEWAY_URL + AI_MODE=relay
                                    ↓
                            重启本地Docker → relay_client自动启动
                                    ↓
                            小程序登录 → 享受全部专业版功能
```

**降级路径**: 用户取消订阅 → 网关JWT过期 → relay_client断开 → 本地Docker自动降级为基础版模式（基础功能不受影响）
