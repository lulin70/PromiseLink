| 维度 | 工具 | 现状 | 目标 |
|------|------|------|------|

### 测试经理必须确保
1. **E2E 用真实后端**: 批次 3.3 把 20 个 mock 改真实后端
2. **真实 LLM pipeline E2E**: 批次 2.8 新增 `test_real_pipeline_e2e.py`

6. **真实负载测试**: 批次 3.8 补并发/P95/P99 断言

### 测试质量铁律
- 禁止 `test.skip()`(用户哲学)
- 禁止 `pytest.skip` 除非测试本身逻辑错误(应直接删除)
- 禁止 `xfail strict=False`(隐藏 XPASS)
- 禁止 stub pipeline 为 `pass`(必须有至少 1 个真实 LLM 测试)
- 禁止 "0 结果 = pass" 弱断言
- 禁止 `assertTrue` 绕过(用 `assertEqual/assertIn`)
- 必须覆盖 Happy/Error/Boundary 三维度
