### ⚠️ 遗漏项(优先级:高)

   - 第一期应为手动录入,但 PRD 未明确
   - **缺失**:数据录入界面、字段定义、录入流程

4. **权限管理缺失**

## 四、风险识别

### 🔴 高风险

1. **合规风险**
   - 竞对数据来源未限定,可能违反反爬虫协议或数据使用条款
   - **缓解**:明确仅手动录入公开信息,禁止自动化采集

2. **数据质量风险**
   - 实体归一无人工确认,错误归一难以撤回
   - **缓解**:第一期必须包含人工审核 + 撤回功能

3. **范围蔓延风险**
   - 第一期功能过多(活动管理 + 实体归一 + 竞对分析 + 校友网络)
   - **缓解**:建议第一期聚焦活动管理 + 基础实体归一,其他功能降级为原型

### 🟡 中风险

4. **用户体验风险**
   - 校友网络功能依赖手动录入,初期数据稀疏
   - **缓解**:设定最小可用数据量(如 50 条校友记录)

5. **技术债务风险**
   - 第一期"快速实现"可能导致架构不可扩展
   - **缓解**:明确第一期需预留扩展点(如实体归一规则引擎接口)
