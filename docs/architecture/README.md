## 🏗️ 架构概览

### 三层架构模型

```
┌─────────────────────────────────────────┐
│  L1: 应用层（许总团队 - IAMHERE名片）   │
│  · 名片小程序                             │
│  · H5页面（WebView内嵌）                 │
└──────────────┬──────────────────────────┘
               │ HTTP API
┌──────────────▼──────────────────────────┐
│  L2: 引擎层（CarryMem - PromiseLink）     │
│  · 事件标准化                             │
│  · 语义路由（4条管线）                    │
└──────────────┬──────────────────────────┘
               │ Message Bus
┌──────────────▼──────────────────────────┐
│  L3: 核心引擎（3个独立引擎）              │
│  · 实体归一引擎                           │
│  · 关联发现引擎                           │
│  · Todo生成引擎                          │
└─────────────────────────────────────────┘
```

### 技术栈

| 层级 | 技术选型 |
|------|---------|
| **API框架** | FastAPI (Python 3.11+) |
| **数据库** | PostgreSQL 15 (生产) / SQLite (开发) |
| **缓存** | Redis 7 |
| **图计算** | NetworkX + igraph |
| **LLM** | Moka AI (Claude Sonnet) + spaCy |
| **容器化** | Docker + docker-compose |

## 🔑 核心设计

### 1. 数据模型（§3）

#### 4张核心表
1. **Event**：事件表（4种类型：card_save/meeting/call/manual）
2. **Entity**：实体表（5种类型：person/organization/topic/technology/project）
3. **Association**：关联表（8种关联类型）
4. **Todo**：待办表（6种类型 + 5种状态）

### 2. P0核心算法（§4）

#### 实体归一5步算法（§4.4，第616-718行）
```
Step 1: 精确匹配（规范化+hash）
Step 2: 别名匹配（预定义+用户自定义）
Step 3: 模糊匹配（RapidFuzz，阈值0.85）
Step 4: 上下文匹配（同事件实体共现）
Step 5: 人工确认 + 撤回机制
```

#### 商机匹配度六维算法（§4.5，第724-857行）
```
Score = keyword_overlap × 0.25
      + industry_alignment × 0.20
      + topic_similarity × 0.15
      + llm_semantic × 0.10
      + interaction_history × 0.10
      + callability_score × 0.20
```

#### Todo状态机（§4.6，第860-941行）
```
pending ⇄ in_progress ⇄ done
   │                      │
   └──── dismissed ───────┘
   │
   └──── snoozed ────→ (定时恢复)
```

### 3. API设计（§5）

#### 核心端点
- `POST /api/v1/events` - 创建事件
- `GET /api/v1/entities` - 查询实体
- `GET /api/v1/associations` - 查询关联
- `GET /api/v1/todos` - 查询Todo
- `POST /api/v1/todos/{id}/feedback` - 反馈闭环

### 4. 安全设计（§6）

- **传输加密**：TLS 1.3 + HSTS
- **身份认证**：JWT（RS256，access:15min, refresh:7d）
- **权限控制**：JWT认证 + 单用户数据隔离（无RBAC）
- **数据加密**：AES-256-GCM字段级加密
- **审计日志**：全写操作记录
