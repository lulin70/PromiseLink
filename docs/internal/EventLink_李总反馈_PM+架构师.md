## 产品经理(PM)
# PromiseLink 产品定位升级与MVP优化方案

## 1. 定位升级评估：完全同意

**理由：**
- 市场差异化：扫名片是红海（名片全能王等成熟产品），商务关系管理是蓝海
- 用户痛点更真实：商务人士核心需求不是"存名片"而是"记住聊了什么、下次见面聊什么"
- AI价值最大化：从OCR识别（低价值）转向内容理解+行动建议（高价值）
- 商业模式更清晰：关系管理是持续付费场景，工具是一次性需求

## 2. MVP-Core功能调整（聚焦4个核心）

**保留强化：**
1. **快速建档**（原名片扫描升级）：支持拍名片/手动输入/语音速记，3秒建档
2. **智能商机识别**：从会话记录提取需求、资源、时间窗口
3. **行动提醒**：基于商机自动生成跟进任务

**新增：**
4. **会话速记**：见面后语音/文字快速记录"聊了什么"（核心场景）

**Phase 2延后：**
- 关系图谱可视化（非MVP必需）
- 批量导入（违背"真实接触"理念）
- 高级搜索（数据量小时无价值）

## 3. 冷启动解决方案（5个可执行）

**方案1：首次引导模板**
- 预置3个场景卡片："刚认识的潜在客户""行业活动新朋友""供应商对接人"
- 用户选场景→填3个关键信息（姓名+公司+一句话需求）→立即生成示例商机

**方案2：微信聊天导入**
- 授权读取最近7天微信聊天（仅标题/摘要）
- AI提取可能的商务联系人，用户确认后一键建档

**方案3：日历会议同步**
- 对接Outlook/飞书日历，识别外部参会人
- 会后弹窗："刚和张三开完会，快速记录聊天要点？"

**方案4：语音极速模式**
- 打开APP直接语音："今天见了腾讯的李四，他们Q3要采购视频会议系统"
- 自动拆解为联系人+商机+提醒

**方案5：邀请奖励机制**
- 邀请3个好友各建1条记录，解锁"关系图谱"功能

## 4. Phase 1验证方案

**商机准确性（2周快速验证）：**
- 召集20个种子用户，每人记录5次真实商务接触
- 对比"AI生成的商机"vs"用户手动标注的商机"，准确率目标≥70%
- 失败case人工复盘，优化prompt

**隐私信任（设计先行）：**
- 数据本地存储优先（Phase 1仅云端备份加密数据）
- 设置页明确展示："您的数据仅用于个人助手，不会被分享、训练模型或商业化"
- 敏感信息标记功能："这条记录包含商业机密，不参与AI分析"

## 5. PRD产品概述重写

**旧版：** PromiseLink是智能商务社交平台，通过AI技术帮助用户管理人脉关系

**新版：**
PromiseLink是商务关系管理AI助手。帮您把每次见面、每通电话、每个可能的合作机会，自动整理成清晰的待办事项。不用再担心"上次和他聊到哪了""这个人能帮什么忙"——PromiseLink记得比您清楚，并在合适的时机提醒您行动。

**核心场景：** 展会认识50人→语音速记关键信息→AI识别3个高优先级商机→一周内完成精准跟进

## 6. 数字名片小程序对接POC方案

**技术方案（最小化集成）：**
1. **Webhook触发：** 小程序扫码后POST联系人JSON到PromiseLink API
2. **数据映射：** `{name, company, title, phone, wechat}` → PromiseLink Contact模型
3. **智能补全：** 弹窗询问："刚扫了张三的名片，你们主要聊了什么？"用户补充→触发商机识别
4. **双向同步（可选）：** PromiseLink更新联系人备注→回写到小程序（需小程序开放API）

**POC验证目标：** 1周内完成扫码→建档→补充会话→生成商机全流程，跑通10个真实case

**总结：** 定位升级是战略正确决策。MVP聚焦"快速建档+会话速记+商机识别+行动提醒"4个功能，用语音速记+微信导入解决冷启动，2周内用20个种子用户验证商机准确性。

## 架构师(Architect)
# PromiseLink 架构评估与调整建议

## 1. 同意定位升级吗？为什么？

**同意。** 理由：
- 名片识别是低频场景（线下会议），商务沟通是高频场景（每天都在发生）
- "聊了什么→下一步做什么"是真实痛点，有留存价值
- AI助手定位更容易产生粘性和数据积累，名片工具用完即走
- 技术可行性更强：LLM擅长对话分析和行动提炼，不依赖OCR准确率

## 2. MVP-Core 7个功能需调整吗？

**建议调整为5个核心功能：**

**保留并强化：**
- 快速建档（改为"多渠道录入"：语音/文字/名片都可以）
- 商机识别（聚焦"下一步行动"而非静态画像）
- 智能提醒（这是留存关键）

**合并优化：**
- 名片扫描→降级为建档的一种方式，不独立成功能
- 关系图谱+人脉搜索→合并为"关系洞察"（Phase 2再做可视化）

**新增：**
- 沟通记录（核心！微信对话/会议纪要快速导入）

## 3. 冷启动如何解决？

## 共识结论

# PromiseLink 产品定位与MVP方案共识总结

## 1. 产品定位最终定义

**从「智能名片管理工具」升级为「商务关系管理AI助手」**

**核心转变：**
- **从低频工具到高频助手**：从"扫名片存档"转向"记录每次沟通+提炼行动"
- **从被动存储到主动赋能**：不只是记住"是谁"，更要理解"聊了什么、下一步做什么"
- **从红海竞争到蓝海创新**：避开名片工具红海，切入商务关系管理未被满足的需求

**一句话定位：**
把每次商务接触自动转化为可执行的下一步行动，让商机不再遗漏。

## 4. PRD产品概述重写

### 产品概述
**PromiseLink - 商务关系管理AI助手**

帮助商务人士把每次见面、每段对话、每个潜在合作机会，自动转化为清晰的下一步行动。

## 5. POC技术方案

### 5.1 商机准确性验证（2周）

**执行方案：**
1. 招募20个种子用户（销售/BD/投资人各1/3）
2. 每人记录5次真实商务接触（语音/文字不限）
3. 对比"AI生成商机"vs"用户手动标注商机"

**成功指标：**
- 高置信度建议准确率 ≥ 70%
- 用户"有用"反馈率 ≥ 60%
- 失败case人工复盘，优化prompt

**技术实现：**
- 每条建议带置信度标签（高/中/低）
- 用户点击"有用/无用"反馈
- 后台记录准确率数据

### 5.2 隐私信任方案

**技术措施：**
- 数据本地化：提供私有部署版Docker镜像（敏感客户）
- 透明化：每条数据显示"AI分析了哪些内容"
- 可控性：一键删除+AI遗忘功能
- 加密存储：云端备份全程加密

**产品声明：**
- 设置页明确："您的数据仅用于个人助手，不会被分享、训练模型或商业化"
- 敏感信息标记："这条记录包含商业机密，不参与AI分析"

### 5.3 数字名片小程序对接POC

**推荐方案：Webhook回调**
