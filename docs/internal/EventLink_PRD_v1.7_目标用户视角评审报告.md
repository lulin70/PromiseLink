## 产品经理(PM)

# EventLink PRD 用户视角评审

## 1. 我最想要但PRD里没有的功能

**P0 - 关系强度可视化**
- **用户心声**："我想一眼看出谁是我的核心人脉，谁只是泛泛之交"
- **缺失**：PRD只有关联发现，没有人脉网络的可视化呈现（关系图谱、亲密度评分、互动频次）
- **建议**：增加"人脉地图"页面，用节点大小表示关系强度，颜色区分关系类型，点击查看互动历史

**P1 - 批量导入历史联系人**
- **用户心声**："我微信有3000个好友，难道要一个个扫名片？"
- **缺失**：PRD只提到扫名片/录音/会议，没有存量数据导入方案
- **建议**：支持从微信/企业微信/LinkedIn/通讯录批量导入，AI自动补全信息

**P1 - 跟进提醒和自动化**
- **用户心声**："Todo生成了，但我忙忘了，能不能自动提醒我？"
- **缺失**：PRD的Todo是静态列表，缺少时间维度的主动触达
- **建议**：增加智能提醒（会前1小时/承诺到期前1天），支持设置跟进周期（如"每月联系一次"）

## 2. 我用的时候最可能卡住的地方

**P0 - 冷启动问题**
- **卡点**："刚装App，系统说没有关联，因为我只有1个联系人"
- **原因**：PRD未说明最少需要多少数据才能产生价值
- **建议**：设计新手引导，前10个联系人给予"潜在价值预览"（如"再添加5人可能发现校友关系"），或接入公开数据（天眼查/LinkedIn）做冷启动

**P1 - 误报处理**
- **卡点**："系统说A和B是竞对，但其实不是，我怎么纠正？而且下次还会推送"
- **原因**：PRD的US-08只提到归一确认,未提及关联纠错机制
- **建议**：所有Todo支持"标记错误"并说明原因,系统学习后不再推送同类关联

**P2 - 隐私顾虑**
- **卡点**："录音会议都上传,我的商业机密会不会泄露？"
- **原因**：PRD安全章节偏技术,用户看不懂
- **建议**：首页显著位置用大白话说明"数据只存你的私有空间,我们看不到"

## 3. 我会拿这个产品和____比较

**vs. 脉脉/LinkedIn - 缺了社交互动**
- **对比发现**：竞品有动态/点赞/评论,EventLink只是单向记录
- **建议**：Phase 2增加"轻社交"功能,如给联系人发祝福/分享文章,增加粘性

**vs. Notion/飞书文档 - 缺了知识沉淀**
- **对比发现**：PRD提到"智慧大脑",但没有笔记/标签/搜索能力
- **建议**：会议记录支持手动补充笔记,按项目/话题归档,全文搜索

**vs. 销售易/纷享销客CRM - 缺了团队协作**
- **对比发现**：竞品支持线索分配/销售漏斗/团队共享,EventLink是个人工具
- **建议**：Phase 3考虑企业版,支持团队共享人脉(带权限控制)

## 4. 一句话总结

PRD的AI能力很强,但**缺少"用户自己动手"的空间**——无法批量导入、无法可视化人脉、无法纠正错误、无法沉淀知识。建议Phase 1先解决冷启动和误报处理(P0),再逐步补齐可视化和协作能力。

## 2. 用的时候最可能卡住的地方

**🚫 卡点1：扫完名片后不知道"然后呢"**
- PRD说"5秒推送Todo"，但我扫完名片站在对方面前，盯着手机等5秒很尴尬
- **问题**：没有"扫描中"的即时反馈，不知道是在处理还是卡死了
- **建议**：扫描后立即显示"正在分析关联..."进度条，3秒内先显示基础信息（公司/职位），关联发现异步加载
- **严重程度**：P0（破坏第一印象）

**🚫 卡点2：Todo太多不知道先做哪个**
- PRD有⚪🔵🔴三种类型，但没说优先级排序逻辑
- **问题**：打开App看到20条Todo，全是平铺列表，不知道哪个紧急
- **建议**：默认按"时间敏感度"排序：今日会议相关>本周跟进>长期机会；增加"只看今日"快捷筛选
- **严重程度**：P0（信息过载导致放弃使用）

**🚫 卡点3：不敢相信AI的判断但又没法验证**
- PRD说"竞对关系预警"，但我不知道AI是怎么判断的
- **问题**：看到🔴风险提示"该公司与你的客户是竞对"，但没有证据链，不敢采取行动
- **建议**：每个关联卡片必须有"查看依据"入口，展开显示：数据来源（哪次会议/哪张名片）+推理路径（因为A→所以B）
- **严重程度**：P1（信任度问题）

## 3. 会拿来比较的产品和缺失功能

**📱 对比对象1：脉脉/LinkedIn**

- **建议**：关联卡片上显示共同联系人头像墙（最多5个），点击查看完整列表
- **严重程度**：P1（社交产品标配）

**📱 对比对象2：Notion/滴答清单**
- **缺失**：Todo没有"拖拽改期"和"批量操作"

- **建议**：Todo列表支持长按多选→批量标记完成/延期/删除；支持拖拽到日历改期
- **严重程度**：P2（效率优化）

**📱 对比对象3：销售易/纷享销客CRM**
- **缺失**：没有"客户画像"聚合视图
- CRM会把一个客户的所有信息聚合成一页，EventLink的关联是碎片化的Todo
- **建议**：增加"人物档案"页：聚合该联系人的所有关联、历史互动、待办事项、话题标签
- **严重程度**：P1（B端用户习惯）

## 安全专家(Security)

# EventLink 安全视角评审（目标用户视角）

## 1. 最想要但PRD里没有的功能

**P0 - 数据导出与删除权**
作为商务人士，我最担心的是"数据被锁死在你们系统里"。我需要：
- 一键导出我的所有名片、关系图谱、会议记录（标准格式如vCard/JSON）
- 彻底删除账号和所有数据的能力（不是"停用"，是真删除）
- 删除前能预览"哪些关联会断掉"

**P0 - 敏感信息脱敏控制**
录音和会议记录里经常涉及报价、合同条款、未公开项目。我需要：
- 上传前自动识别敏感词（金额/项目代号/客户名）并打码
- 手动标记"这段别给AI分析"的能力
- 分享名片时隐藏特定字段（如私人手机号只给内部看）

## 2. 使用时最可能卡住的地方

**P0 - 不知道AI看了我哪些数据**
PRD说"自动采集录音/会议"，但我不知道：
- 哪些会议被录了？能看到录音列表吗？
- AI分析时读取了我通讯录的哪些人？
- 关联发现用了我几年前的旧数据吗？

**卡点**：第一次用就担心"是不是把我所有隐私都扫描了"，不敢继续。

**P1 - 误推送让我社死**
系统推"🔴竞对预警"，但万一是误报呢？我拿着这个去质疑客户，关系崩了谁负责？

**卡点**：不敢信任AI判断，宁可不用风险预警功能。

## 3. 会拿来比较的产品和缺失

**对比对象：脉脉/LinkedIn + 企查查**

发现EventLink缺了：
- **P0 - 数据来源标注**：脉脉会告诉我"这个关系是从你领英导入的"，EventLink的关联是AI推测的还是真实数据？没说清楚
- **P1 - 关联可信度分级**：企查查会标"疑似关联/已确认关联"，EventLink只有置信度数字，我看不懂0.82是"可能"还是"确定"
- **P2 - 黑名单功能**：我不想让系统分析某些人（如前任/竞争对手），但PRD没提供屏蔽名单

## 4. 严重程度汇总

| 问题 | 等级 | 理由 |
|------|------|------|
| 数据导出删除权 | P0 | 无此功能直接违反GDPR/个保法，企业客户不敢用 |
| 敏感信息脱敏 | P0 | 商务场景必然涉及机密，无脱敏=泄密风险 |
| 数据透明度 | P0 | 用户看不到AI用了啥数据=信任崩塌 |

- **场景**：系统问我"张三(ID:1234)和张三(ID:5678)是同一人吗？"，我看到一堆JSON格式的属性对比，不知道怎么判断
- **修改建议**：用自然语言显示"两个张三都是：某公司CEO / 手机尾号1234 / 2023年认识 → 可能是同一人"

### 2.3 **离线后不知道数据有没有同步** (P1)
- **卡点**：PRD说"离线时本地缓存"，但我不知道什么时候会同步、同步失败了怎么办
- **场景**：地铁里扫了5张名片，出地铁后不知道有没有上传成功
- **修改建议**：状态栏显示"⏳ 5条待同步"，同步完成后显示"✓ 已同步"

---

## 3. 用户会拿来比较的产品和缺失功能

### 3.1 **对比微信/企业微信** (P0)
- **用户期待**：微信里可以直接发消息、打电话、看朋友圈
- **EventLink缺失**：扫完名片后没有"立即联系"的快捷入口
- **修改建议**：Todo卡片增加"📱 拨打电话 / 💬 发微信 / ✉️ 发邮件"快捷按钮

### 3.2 **对比脉脉/LinkedIn** (P1)
- **用户期待**：能看到对方的职业轨迹、公司动态、共同好友
- **EventLink缺失**：只有静态名片信息，没有动态更新和社交关系可视化
- **修改建议**：Phase 2增加"时间线视图"，显示"张三 2020-2022在A公司 → 2023至今在B公司"

### 3.3 **对比Notion/飞书文档** (P2)
- **用户期待**：Todo可以关联文档、附件、会议纪要
- **EventLink缺失**：Todo只有文字描述，无法附加资料
- **修改建议**：Todo支持"📎 添加附件 / 🔗 关联文档"

---

## 4. 严重程度总结

| 问题 | 严重程度 | 影响范围 |
|------|---------|---------|
| 名片扫描即时反馈 | P0 | 所有用户每次使用 |
| 关联发现解释 | P0 | 核心价值感知 |
| Todo查看时机提醒 | P0 | 信息错过率高 |
| 立即联系快捷入口 | P0 | 闭环缺失 |
| 批量导入 | P1 | 新用户冷启动 |
| 实体归一确认体验 | P1 | 数据质量依赖 |
| 离线同步状态 | P1 | 用户信任感 |
| 职业轨迹可视化 | P2 | 增值功能 |

**总结**：PRD技术架构完整，但**用户操作闭环**和**即时反馈**严重不足，建议Phase 1优先补齐P0问题。

## DevOps(Infra)

## 从目标用户视角的关键遗漏点

### 1. **最想要但PRD里没有的功能**

**P0 - 微信生态集成缺失**
- **用户视角**：商务BD/投资人90%的社交发生在微信，但PRD只提到"数字名片/录音/会议"，没有微信聊天记录、朋友圈、公众号文章的信息采集
- **卡点**：扫完名片加了微信后，后续所有沟通都在微信，系统无法追踪，关联发现就断了
- **修改建议**：Phase 1至少支持手动粘贴微信聊天记录上传；Phase 2考虑企业微信API对接

**P1 - 移动端离线能力不足**
- **用户视角**：商务场景常在地铁/电梯/会议室等弱网环境，PRD提到"离线策略"但未说明名片扫描、Todo查看等核心功能是否离线可用
- **卡点**：客户面前掏出手机扫名片，转圈10秒加载不出来，非常尴尬
- **修改建议**：明确"名片OCR+本地缓存+后台同步"的离线优先架构，关键操作3秒内必须有反馈

---

### 2. **使用时最可能卡住的地方**

**P0 - 冷启动数据稀疏问题**
- **用户视角**：新用户前100张名片时，系统几乎发现不了关联（数据太少），看不到价值就流失了
- **卡点**：PRD强调"自动发现关联"，但没说如何解决冷启动期的空窗期
- **修改建议**：
  - 允许导入通讯录/LinkedIn/脉脉等存量数据快速建图
  - 前50张名片提供"手动标注关系"引导，用户教系统的同时建立初始图谱
  - 冷启动期降低关联阈值(0.6→0.4)，宁可多推也别让用户觉得"没用"

**P1 - Todo泛滥导致麻木**
- **用户视角**：每扫一张名片就推3-5条Todo(⚪🔵🟢)，一天扫20张就是60-100条，根本看不过来
- **卡点**：PRD没有Todo优先级排序、智能聚合、批量操作的设计
- **修改建议**：
  - Todo按"紧急度×价值"排序，默认只显示Top 10
  - 同类Todo自动聚合(如"5个商机匹配待破冰")
  - 支持"全部标记已读""稍后提醒"等批量操作

---

### 3. **会拿来比较的竞品及缺失功能**

**P0 - 对比脉脉/LinkedIn：缺少公开信息自动补全**
- **用户视角**：脉脉扫名片后自动拉取职业履历、教育背景、共同好友，EventLink只有用户手动输入的信息
- **缺失**：PRD没有"公开数据源补全"功能(如企查查API、天眼查API、领英爬取)
- **修改建议**：Phase 1对接1-2个免费API(如企查查基础版)，自动补全公司信息、融资阶段

**P1 - 对比销售易/纷享销客：缺少CRM核心功能**
- **用户视角**：商务BD需要记录"跟进阶段/赢单概率/合同金额"，EventLink只有关联发现，没有销售管理
- **缺失**：客户分级、销售漏斗、跟进记录、合同管理等CRM基础能力
- **修改建议**：Phase 2增加"轻量级销售管理"模块，至少支持客户分级(A/B/C)和跟进记录

**P2 - 对比Notion/飞书文档：缺少知识沉淀**
- **用户视角**：会议纪要、行业洞察、客户画像需要长期积累，PRD只提到"事件采集"，没说如何沉淀为可检索的知识库
- **缺失**：标签体系、全文搜索、知识图谱可视化
- **修改建议**：Phase 2增加"知识中心"，支持按人/公司/话题聚合历史事件，形成客户360°画像

---

### 4. **其他高危遗漏**

**P0 - 隐私合规风险**
- PRD提到"录音采集"但未说明用户知情同意机制，商务场景录音涉及法律风险
- **修改建议**：录音前强制弹窗"已告知对方并获得同意"的确认框，并在设置中提供"录音水印"(语音开头自动播报"本次通话正在录音")

**P1 - 多设备同步策略缺失**
- 商务人士常用"手机扫名片+电脑看报告"，PRD未说明跨设备数据同步逻辑
- **修改建议**：明确"实时云同步+冲突解决策略"，关键操作5秒内同步到所有设备

---

**总结**：PRD技术架构完整，但**用户实际使用场景的最后一公里**考虑不足，尤其是微信生态、冷启动、Todo管理、竞品对标四个维度需要补强。建议优先解决P0问题，否则用户留存率会很低。
