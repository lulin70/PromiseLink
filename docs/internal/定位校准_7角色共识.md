# 定位校准后7角色共识调整 — 「自用的私密资源经营助手」

> 日期: 2026-06-02
> 触发: 用户关键纠正「PromiseLink不是一个资源撮合的平台，而是一个自用的私密资源经营助手」
> 基于文档: 李总资源匹配建议 + WorkBuddy审阅分析 + 7角色原始共识(2400+行)

---

## 📌 TL;DR：定位校准的核心影响

| 维度 | 之前共识（平台视角） | 校准后（私密助手视角） | 变化 |
|------|---------------------|---------------------|------|
| 产品定位 | AI资源关系经营与匹配**平台** | AI私密资源经营**助手** | ⬇️ 降级平台属性 |
| 数据归属 | 资源需公开/授权共享 | 全在你自己手里 | ⬇️ 零隐私压力 |
| 匹配逻辑 | A的供给匹配B的需求 | **你的需求**匹配**你人脉**的供给 | ⬇️ 单边匹配 |
| 冷启动 | 需要双边密度 | 你本来就认识这些人 | ⬇️ 难度大幅下降 |
| RBAC | Phase1必须做 | **明确排除** | ❌ 不需要 |
| 资源授权共享 | Phase2做 | **明确排除** | ❌ 不需要 |
| 他人资源 | 可录入+可匹配 | **不可匹配，仅做笔记** | ❌ 安全红线 |
| 团队协作 | Phase2规划 | **明确排除** | ❌ 不需要 |
| 资源敏感度 | 4级(public/internal/private/confidential) | 2级(可匹配/不匹配) | ⬇️ 简化 |
| 审计日志 | 全量审计 | 仅记录操作历史 | ⬇️ 简化 |

| DevOps | PoC五维→六维 | ✅ **PoC五维→六维** | ➡️ 不变 |
| UI设计师 | PoC五维→六维 | ✅ **PoC五维→六维** | ➡️ 不变 |

**跨角色共识**：6/7维持六维方案，但维度含义需要调整：

**校准后的六维匹配算法**：

| 维度 | 权重 | 私密助手下的含义 | 与之前区别 |
|------|------|-----------------|-----------|
| 资源类型匹配 | 25% | 你的人脉有什么类型的资源 | 不变 |
| 能力匹配 | 20% | 资源能力是否解决你的需求 | 不变 |
| 条件匹配 | 15% | 调用条件是否满足 | 不变 |
| **可调用度** | **20%**（↑5%） | 你与资源持有人的关系是否支持调用 | **权重提高**：私密助手下这是核心维度 |
| 关系强度 | 10% | 你和这个人的关系深度 | 不变 |
| 历史合作 | 10% | 过去调用结果如何 | 不变 |

**关键调整**：「资源可调用度」权重从15%→20%，因为私密助手下匹配逻辑是「你的需求匹配你人脉的供给」，可调用度是最关键的判断维度。

## 二、平台级功能的明确排除清单

### ❌ 明确排除（私密助手不需要）

| 功能 | 之前共识阶段 | 排除理由 |
|------|------------|---------|
| RBAC权限模型 | Phase1规划 | 私密助手只有1个用户，不需要角色权限 |
| 资源授权共享 | Phase2规划 | 私密助手下资源不共享给他人 |
| 团队协作 | Phase2规划 | 私密助手是个人工具，不是团队工具 |
| 他人可提供资源匹配 | Phase1规划 | 私密助手下只能匹配自己人脉的资源 |
| 资源授权申请流程 | Phase1规划 | 不存在"向资源方申请授权"的场景 |
| resource_permissions表 | Phase1设计 | 不需要资源级权限控制 |
| 跨用户资源访问控制 | Phase1安全设计 | 不存在跨用户访问 |
| 多租户隔离 | Phase2设计 | 不需要 |
| 团队资源池/看板 | Phase2设计 | 不需要 |
| 企业管理看板 | Phase3设计 | 不需要 |

### ✅ 保留（私密助手下仍有价值）

| 功能 | 保留理由 |
|------|---------|
| 资源敏感度分级 | 简化为2级：可匹配/不匹配。用户标记某些资源"不想被AI推荐" |
| 需求确认4条规则 | AI提取的信息需用户确认，这是AI代理权限边界 |
| 操作审计日志 | 简化为操作历史记录（非安全审计），帮助用户回溯 |
| LLM输入消毒 | 防Prompt注入仍然需要 |
| 匹配推荐可解释性 | 用户需要知道AI为什么推荐 |
| 资源透支提醒 | "30天向同一人请求3次"在私密助手下更有意义（帮你自己管理关系健康度） |
| 调用结果补录 | 记录引荐结果，更新资源可靠性 |
| 资源供给卡 | 简化版，附着在Person详情页 |
| 需求录入 | 语音/文字一句话录入 |
| 匹配算法 | 六维，聚焦"你的需求匹配你人脉的供给" |

## 三、李总建议在私密助手定位下的价值重估

### 🟢 高价值（私密助手下更有价值）

| 建议 | 理由 |
|------|------|
| 资源供给卡 | 私密助手下"看清谁有什么资源"是核心能力 |
| 需求录入 | "我需要什么"→ 匹配 "我的人脉谁能提供" |
