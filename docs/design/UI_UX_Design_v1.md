## 1. 设计原则

### 1.1 核心原则 [0.2.0更新]
- **私密优先**：所有数据仅属用户本人，无分享、无协作、无他人可见
- **移动优先**：以微信小程序为第一适配目标（375px-428px），单手可操作
- **回应驱动**：首页从"信息展示"转向"回应驱动"，双核心区域引导用户行动
- **信息密度适中**：莫兰迪色系降低视觉疲劳，长时间使用不刺眼
- **快速操作≤3点击**：核心操作不超过3次点击到达目标
- **微信生态适配**：遵循小程序设计规范

### 1.2 明确排除 [0.2.0更新]
- ❌ RBAC / 多租户 / 团队协作
- ❌ 他人资源匹配（仅匹配自己的人脉资源）
- ❌ 分享功能（私密助手，不分享任何数据）
- ❌ 原生APP（微信小程序为主入口 + H5 WebView备选）
- ~~❌ 人脉推荐~~ → **[0.2.0] 已正式移除**：与"私密助手"定位冲突，不做主动推荐（D6-9）

### 1.3 关键术语 [0.2.0更新]
| 术语 | 说明 | 字段名 |
|------|------|--------|
| Todo类型 | 6种：cooperation_signal/risk/care/promise/followup/help | `todo_type` |
| 动作类型 | 6种：my_promise/their_promise/my_followup/mutual_action/system_reminder/unclear | `action_type` |
| 可联系度 | 人的可联系程度（0.0-1.0） | `callability` |
| 敏感度 | 2级：matchable / no_match | `sensitivity` |
| 关系阶段 | 7阶段枚举（PoC仅前3阶段可交互） | `RelationshipStage` |
| 关系推进卡 | F-47 RelationshipBrief，12模块全貌视图 | `relationship_briefs` |

## 2. 视觉规范

### 2.1 色彩系统

#### 2.1.1 莫兰迪主色系（Todo类型色）

产品核心视觉语言采用莫兰迪色系——低饱和度、灰调柔和，传达私密助手的温和专业感。6种Todo类型各对应一种莫兰迪色，贯穿整个产品视觉体系。

| 色名 | 色值 | Todo类型 | 语义 | 色板 |
|------|------|----------|------|------|
| 雾白 | `#B8C4C0` | cooperation_signal（合作信号） | 合作机会、可能性 | 🟩 |
| 烟粉 | `#C4A7A0` | risk（风险） | 警示、需关注 | 🟫 |
| 雾蓝 | `#A0B0C4` | care（关心） | 关心、关注 | 🟦 |
| 雾绿 | `#A0C4A8` | promise（承诺） | 承诺、兑现 | 🟩 |
| 雾金 | `#C4C0A0` | followup（跟进） | 跟进、待定 | 🟨 |
| 雾紫 | `#B0A0C4` | help（帮助） | 帮助、支持 | 🟪 |

[0.2.0新增] **关系推进卡模块色标**：

| 模块类别 | 推荐色值 | 使用场景 |
|---------|---------|---------|
| 阶段标识 | 雾绿 `#A0C4A8` | 当前关系阶段进度条、阶段标签 |
| 对方关注 | 雾蓝 `#A0B0C4` | concerns模块标签 |
| 承诺事项 | 雾绿 `#A0C4A8` | active_promises模块 |
| 需求洞察 | 雾白 `#B8C4C0` | need_insights模块 |
| 帮助贡献 | 雾紫 `#B0A0C4` | contribution/help模块 |
| 下一动作 | 雾金 `#C4C0A0` | next_natural_action模块 |

**使用规范**：
- 主色用于Todo类型标识（左侧色条、标签背景、图标着色）
- 主色作为背景时，文字使用深色（`#3A3F42`）确保可读性
- 主色作为标签时，使用10%透明度背景 + 100%边框

#### 2.1.2 功能色

功能色复用莫兰迪色系，保持视觉统一性。

| 功能 | 色值 | 来源 | 使用场景 |
|------|------|------|---------|
| 成功 | `#A0C4A8` | 雾绿 | 操作成功提示、完成状态 |
| 警告 | `#C4C0A0` | 雾金 | 警告提示、待处理状态 |
| 错误 | `#C4A7A0` | 烟粉 | 错误提示、风险状态 |
| 信息 | `#A0B0C4` | 雾蓝 | 信息提示、背景说明 |

#### 2.1.3 中性色

| 用途 | 色值 | 说明 |
|------|------|------|
| 主文字 | `#3A3F42` | 标题、重要信息 |
| 次文字 | `#6B7280` | 正文、描述 |
| 辅助文字 | `#9CA3AF` | 时间戳、标签 |
| 占位文字 | `#C4C8CC` | 输入框placeholder |
| 分割线 | `#DDE0E3` | 列表分割、区域分隔 |
| 页面背景 | `#F5F6F7` | 全局背景色 |
| 卡片背景 | `#FFFFFF` | 卡片/弹窗背景 |
| 禁用背景 | `#EDF0F2` | 禁用状态/PoC不可用阶段 |
| 禁用文字 | `#C4C8CC` | 禁用状态文字/PoC不可用阶段文字 |

#### 2.1.4 色彩使用规范

**对比度要求**：
- 主文字（`#3A3F42`）在白色背景上：对比度 ≥ 12:1 ✅
- 次文字（`#6B7280`）在白色背景上：对比度 ≥ 5.5:1 ✅
- 莫兰迪主色在白色背景上：对比度约 2.5-3.0:1，**仅用于装饰性色条/图标**

**色盲友好**：
- 不单独依赖颜色传达信息，必须配合图标/文字标签
- Todo类型标识 = 颜色色条 + 图标 + 文字标签，三重编码

### 3.8 🆕 RelationshipProgressBar（关系阶段进度条）[0.2.0新增]

7阶段水平进度条组件，可视化RelationshipStage。

```
┌─────────────────────────────────────────────────────┐
│  关系阶段                                            │
│  ○───○───●───○───○───○───○                         │
│  新认识 了解需求 价值回应 合作探索 意向确认 执行中 复盘 │
│        ↑ 当前阶段                                    │
│              [确认升级到：合作探索]                    │
└─────────────────────────────────────────────────────┘
```

**属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| current_stage | enum | 当前阶段（RelationshipStage枚举） |
| stages | array | 7个阶段的完整定义 |
| interactive_stages | int | 可交互阶段数（PoC=3） |
| can_advance | bool | 是否可升级到下一阶段 |
| next_stage | enum | 建议的下一阶段 |

**视觉规范**：
- 进度条水平排列，每个阶段 = 圆形节点(20px) + 连线 + 阶段名称(10px)
- 已完成阶段：节点填充雾绿`#A0C4A8`，连线实线雾绿
- 当前阶段：节点填充雾绿 + 外圈发光效果(2px雾绿)，阶段名称加粗
- PoC可交互阶段（当前及之前）：正常色彩
- PoC不可交互阶段（后4阶段）：节点和文字均为禁用色(`#C4C8CC`/`#EDF0F2`)，表示"尚未解锁"
- RS-01强制确认：阶段升级按钮仅在AI建议升级时出现，需用户主动点击确认

**PoC阶段特殊表现**：
- 仅前3阶段（新连接→了解需求→价值回应）可交互
- 后4阶段显示为灰色锁定态，hover时提示"该阶段将在后续版本开放"
- 升级按钮文案："确认升级到：{下一阶段名}"

### 3.9 🆕 RelationshipBriefCard（关系推进卡容器）[0.2.0新增]

F-47关系推进卡的UI容器，包含12个模块按优先级分层展示。

**属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| person_id | string | 关联人物ID |
| stage | enum | 当前关系阶段 |
| brief_data | object | 推进卡完整数据（12模块） |
| expand_mode | enum | compact(默认P0) / expanded(P0+P1) / full(P0+P1+P2) |

**展开交互**：
- 默认compact模式：仅显示P0首屏4模块
- 点击"展开更多"→expanded模式：显示P0+P1共7模块
- 点击"查看完整推进卡"→full模式或跳转详情页：全部12模块

#### 核心区域1："今天需要我回应的连接"

**信息架构**：

### 11.1 语音入口设计

#### 全局语音按钮（FAB）

悬浮于小程序右下角，低于首要操作层级（不遮挡核心内容）。

| 属性 | 规范 |
|------|------|
| 位置 | 右下角悬浮按钮(FAB)，距底部导航栏56px + 安全区域 |
| 尺寸 | 56px直径（微信小程序FAB标准） |
| 图标 | 麦克风图标 |
| 默认态 | 半透明莫兰迪雾绿 `#A0C4A8`，60% opacity |
| 按下态(录音中) | 实心雾绿 + 脉冲动画环 + "正在聆听..."提示文字 |
| 处理中态 | 加载旋转动画 + "请稍等..."文字 |
| 播放态(TTS) | 波形动画 + "正在回答" |

**状态转换示意**：

```
  默认态                    按下态(录音)              处理中                  播放态
┌──────┐            ┌──────────────┐        ┌──────────────┐       ┌──────────────┐
│  🎤  │   点击     │   🎤 ●●●     │  松开  │   ⏳ 请稍等  │  完成  │   ▬▬▬▬▬▬    │
│ (半透│ ────────→  │ 正在聆听...  │ ────→ │              │ ──→   │  正在回答    │
│  明)  │            │  ╭╮╭╮╭╮脉冲  │        │   (转圈动画)  │       │  ▬▬▬▬▬▬    │
└──────┘            └──────────────┘        └──────────────┘       └──────────────┘
```

#### 页面内语音入口

| 页面 | 入口位置 | 触发行为 | 说明 |
|------|---------|---------|------|
| 首页 | 区域1"今天需要我回应"标题右侧 | 小麦克风图标(24px) | 触发该区域的语音查询（如"今天有什么待办"） |
| 人物详情页(推进卡) | 底部操作栏 | 语音按钮 + 提示文案"问：张总怎么样了" | 针对当前人物的语音问答 |
| Todo列表页 | 顶部搜索条右侧 | 麦克风图标 | 语音搜索/筛选Todo |
| 录入页(/input) | 主输入区中央 | VoiceRecorder组件(56px) | 核心录入入口，语音→ASR→实体提取 |

### 11.2 录音交互流程

#### 完整交互状态机

```
┌──────────┐   ┌──────────────────┐   ┌────────────────┐   ┌──────────┐
│  点击    │   │  按住/触摸录音    │   │  松开/自动停止  │   │  处理中   │
│  麦克风   │ → │  (波形可视化)    │ → │ (15s上限/静音3s)│ → │ (转圈+文案)│
└──────────┘   └──────────────────┘   └────────────────┘   └──────────┘
                                                                   ↓
                                    ┌──────────────────────────────────────┐
                                    │         TTS 播放回答                 │
                                    │  ┌────────────────────────────┐     │
                                    │  │ ▬▬▬▬▬ 音波动画             │     │
                                    │  │ "您今天有2场会议。" (同步文字)│    │
                                    │  │ [⏸ 暂停] [🔄 重播] [✕ 关闭] │     │
                                    │  └────────────────────────────┘     │
                                    └──────────────────────────────────────┘
```

#### 关键交互细节

| 参数 | 值 | 说明 |
|------|-----|------|
| 录音最长时间 | 15秒 | 足够说完一个完整问句 |
| 自动停止条件 | 静音超过3秒 | VAD（语音活动检测）自动结束录音 |
| 取消方式 | 上滑取消 | 类似微信语音消息的交互手势 |
| 录音视觉反馈 | 实时波形可视化 + 剩余时间倒计时 | 波形振幅反映音量大小 |
| 最短有效录音 | 0.5秒 | 过短视为误触，不发送 |

#### 文字降级模式（公共场合/静音场景）

- **自动触发**：检测到环境噪音>阈值时，弹出提示：

- **文字模式界面**：移除波形和TTS控件，直接展示结构化回答卡片（同上"展开详情"部分），保留"🎤 再说一次"入口

### 11.4 错误与异常状态的UI处理

| 场景 | UI表现 | 用户可执行操作 | 技术对应 |
|------|--------|--------------|---------|
| ASR失败（没听清） | "没听清，能再说一次吗？" + 🎤重试按钮 | 点击重试 / 重新录音 | ASR confidence < 阈值 |
| NLU不确定 | "您是想问今天的日程吗？" + [是的][不是，我想问...] | 选择确认 或 重新说 | NLU top intent confidence 低 |
| 无结果 | "没有找到相关信息" + 相关建议链接(2-3条) | 查看 建议 / 换种问法 | 检索结果为空 |
| 网络异常 | "网络有问题，稍后再试" + [重试][查看离线缓存] | 重试 / 查看缓存内容 | 网络不可用 |
| 超时（>8s） | "查询较慢，先看看文字版？" + [好的][继续等待] | 切换文字模式 / 继续等待 | 端到端延迟 > 8s |
| 录音过短 | "录音太短了，请再说一次" + 自动重新进入录音 | 重新录音 | 录音时长 < 0.5s |
| 权限未授权 | "需要麦克风权限才能使用语音功能" + [去设置] | 跳转系统设置页 | 微信麦克风权限 denied |

**错误恢复原则**：每种错误状态都必须提供**至少一条前进路径**，不让用户卡死在错误页。

### 11.5 无障碍优化（核心刚需）

> 许总有视力障碍，以下不是" nice-to-have"，而是**必须实现**的基础能力。

#### 5.5.1 TTS播报速度可调

设置页提供三档选择：

| 档位 | 速度 | 适用场景 |
|------|------|---------|
| 慢速 | 0.8x | 信息量大/复杂内容的首次听取 |
| 正常 | 1.0x | 日常使用（默认） |
| 快速 | 1.3x | 快速浏览/熟悉内容的回顾 |

- 入口：设置页 → "语音设置" → "播报速度"
- 全局生效：影响所有TTS输出

#### 5.5.2 字体放大支持

- 回答文字区域支持系统字体大小设置（微信小程序 `font-size` 响应式）
- 最小字号不低于14px（正文标准）
- 设置页提供"大号字体"开关

#### 5.5.3 高对比度模式

- 支持深色背景（`#1A1D21`）+ 高亮文字（`#E8EAE D`）
- 莫兰迪色系在深色模式下使用更高亮度版本
- 设置页 → "无障碍" → "高对比度"

#### 5.5.4 单手操作保证

所有语音操作可在 **3步内完成**：

```
步骤1: 唤醒（点击FAB/页面内麦克风）→ 步骤2: 说话（按住录音）→ 步骤3: 听回答（TTS自动播放）
```

- FAB位置固定在右下角拇指热区
- 录音按钮尺寸≥56px（远超44px最小触控标准）
- 不需要打字、不需要精准点击小按钮

#### 5.5.5 屏幕阅读器兼容

- 支持 TalkBack（Android）/ VoiceOver（iOS）读出UI状态
- 所有交互元素配备 `aria-label` / `aria-live` 区域
- 状态变化实时播报："正在录音"、"处理中"、"正在播放回答"
- 微信小程序使用 `wx.createInnerAudioContext` 的无障碍属性

#### 5.5.6 触控目标规范

遵循 Apple HIG 和 WCAG 2.1 AA 标准：

| 元素类型 | 最小尺寸 | 说明 |
|---------|---------|------|
| 所有可点击按钮 | 44×44px | Apple HIG 最小值 |
| FAB语音按钮 | 56×56px | 主要操作，加大触控面积 |
| 录音取消区域(上滑) | 全屏上半区 | 大面积拖拽区域 |
| Tab/列表项 | 高度≥48px | 列表项行高 |
