## 1. 设计原则

### 1.1 核心原则
- **私密优先**：所有数据仅属用户本人，无分享、无协作、无他人可见
- **移动优先**：375px-428px为主视口，单手可操作
- **信息密度适中**：莫兰迪色系降低视觉疲劳，长时间使用不刺眼
- **快速操作**：核心操作≤3次点击
- **微信生态适配**：小程序WebView为主要载体

### 1.2 明确排除
- ❌ RBAC / 多租户 / 团队协作
- ❌ 他人资源匹配（仅匹配自己的人脉资源）
- ❌ 分享功能（私密助手，不分享任何数据）
- ❌ 原生APP（仅H5 + 微信小程序WebView）

### 1.3 关键术语
| 术语 | 说明 | 字段名 |
|------|------|--------|
| Todo类型 | 6种：cooperation_signal/risk/care/promise/followup/help | `todo_type` |
| 可联系度 | 人的可联系程度（0.0-1.0） | `callability` |
| 敏感度 | 2级：matchable / no_match | `sensitivity` |
| 匹配算法 | 六维：keyword(25%)+industry(20%)+topic(15%)+llm(10%)+history(10%)+callability(20%) | - |

## 2. 视觉规范

### 2.1 色彩系统

#### 2.1.1 莫兰迪主色系（Todo类型色）

产品核心视觉语言采用莫兰迪色系——低饱和度、灰调柔和，传达私密助手的温和专业感。6种Todo类型各对应一种莫兰迪色，贯穿整个产品视觉体系。

| 色名 | 色值 | Todo类型 | 语义 | 色板 |
|------|------|----------|------|------|
| 雾白 | `#B8C4C0` | cooperation_signal（合作信号） | 合作机会、可能性 | 🟩 |
| 烟粉 | `#C4A7A0` | risk（风险） | 警示、需关注 | 🟫 |
| 雾蓝 | `#A0B0C4` | care（关心） | 关心、关注 | 🟦 |
| 雾绿 | `#A0C4A8` | promise（承诺） | 承诺、兑现 | 🟩 |
| 雾金 | `#C4C0A0` | followup（跟进） | 跟进、待定 | 🟨 |
| 雾紫 | `#B0A0C4` | help（帮助） | 帮助、支持 | 🟪 |

**使用规范**：
- 主色用于Todo类型标识（左侧色条、标签背景、图标着色）
- 主色作为背景时，文字使用深色（`#3A3F42`）确保可读性
- 主色作为标签时，使用10%透明度背景 + 100%边框

#### 2.1.2 功能色

功能色复用莫兰迪色系，保持视觉统一性，避免刺眼的标准色。

| 功能 | 色值 | 来源 | 使用场景 |
|------|------|------|---------|
| 成功 | `#A0C4A8` | 雾绿 | 操作成功提示、完成状态 |
| 警告 | `#C4C0A0` | 雾金 | 警告提示、待处理状态 |
| 错误 | `#C4A7A0` | 烟粉 | 错误提示、风险状态 |
| 信息 | `#A0B0C4` | 雾蓝 | 信息提示、背景说明 |

**功能色使用规范**：
- 成功/错误仅用于Toast提示和状态标记，不作为大面积填充色
- 警告色用于截止日期临近、待确认事项
- 信息色用于辅助说明、帮助文案

#### 2.1.3 中性色

莫兰迪灰色系，用于文字、背景、边框等基础元素。

| 用途 | 色值 | 说明 |
|------|------|------|
| 主文字 | `#3A3F42` | 标题、重要信息 |
| 次文字 | `#6B7280` | 正文、描述 |
| 辅助文字 | `#9CA3AF` | 时间戳、标签 |
| 占位文字 | `#C4C8CC` | 输入框placeholder |
| 分割线 | `#DDE0E3` | 列表分割、区域分隔 |
| 页面背景 | `#F5F6F7` | 全局背景色 |
| 卡片背景 | `#FFFFFF` | 卡片/弹窗背景 |
| 禁用背景 | `#EDF0F2` | 禁用状态背景 |
| 禁用文字 | `#C4C8CC` | 禁用状态文字 |

#### 2.1.4 色彩使用规范

**对比度要求**：
- 主文字（`#3A3F42`）在白色背景上：对比度 ≥ 12:1 ✅
- 次文字（`#6B7280`）在白色背景上：对比度 ≥ 5.5:1 ✅
- 辅助文字（`#9CA3AF`）在白色背景上：对比度 ≥ 3.0:1（仅用于装饰性文字）
- 莫兰迪主色在白色背景上：对比度约 2.5-3.0:1，**仅用于装饰性色条/图标**，不用于文字

**色盲友好**：
- 不单独依赖颜色传达信息，必须配合图标/文字标签
- Todo类型标识 = 颜色色条 + 图标 + 文字标签，三重编码
- 敏感度标识 = 图标 + 文字，颜色为辅助

**色彩层级**：
1. **第一层级**：莫兰迪主色（Todo类型标识，用户最关注的维度）
2. **第二层级**：中性色（文字、背景，信息载体）
3. **第三层级**：功能色（操作反馈，临时性）

## 3. 组件库（14个组件）

### 3.1 PersonCard（人物卡片）

展示人物基本信息 + 资源标签 + 敏感度标识 + 快速操作。

```
┌─────────────────────────────────────┐
│ [头像]  张三                          │
│         阿里巴巴 · AI算法总监           │
│ [AI算法] [CV经验] [NLP]  🔒不可匹配   │
│                              [>]     │
└─────────────────────────────────────┘
```

**属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| name | string | 人物姓名 |
| company | string | 公司 |
| title | string | 职位 |
| avatar | string | 头像URL |
| resource_tags | string[] | 资源标签（来自properties.resource） |
| sensitivity | enum | matchable / no_match |
| callability | float | 可联系度 0.0-1.0 |

**视觉规范**：
- 卡片背景：`#FFFFFF`，圆角12px，阴影E1
- 资源标签：雾蓝背景10% + 雾蓝边框，文字12px `#6B7280`
- 敏感度标识：
  - `matchable`：不显示标识（默认状态，无干扰）
  - `no_match`：显示🔒图标 + "不可匹配"文字，`#9CA3AF`灰色
- 点击整卡跳转人物画像页

### 3.2 TodoItem（待办事项）

展示Todo类型标识 + 状态 + 描述 + 操作按钮。6种类型对应6种莫兰迪色。

```
┌─────────────────────────────────────┐
│ ▎📌  答应张三本周发方案               │
│ ▎     承诺 · 高优先 · 明天截止         │
│ ▎                          [完成][>] │
└─────────────────────────────────────┘
```

**属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| todo_type | enum | cooperation_signal/risk/care/promise/followup/help |
| morandi_color | string | 根据todo_type自动映射的莫兰迪色值 |
| status | enum | pending/in_progress/done/dismissed/snoozed |
| priority | enum | high/medium/low |
| description | string | Todo描述 |
| due_date | datetime | 截止时间 |
| related_entity | object | 关联人物摘要 |

**视觉规范**：
- 左侧色条：4px宽，高度100%，颜色 = morandi_color
- 类型图标：morandi_color着色，16px
- 类型标签：morandi_color 10%背景 + 100%边框，文字12px
- 优先级标记：high=烟粉圆点，medium=雾金圆点，low=雾白圆点
- 已完成状态：文字 `#9CA3AF`，删除线

**morandi_color映射**：
```
cooperation_signal → #B8C4C0（雾白）
risk              → #C4A7A0（烟粉）
care              → #A0B0C4（雾蓝）
promise           → #A0C4A8（雾绿）
followup          → #C4C0A0（雾金）
help              → #B0A0C4（雾紫）
```

## 5. Todo类型视觉规范

### 5.1 类型总览

| todo_type | 中文名 | 莫兰迪色 | 图标 | 语义 |
|-----------|--------|----------|------|------|
| cooperation_signal | 合作信号 | 雾白 `#B8C4C0` | 🤝 | 合作机会、匹配推荐 |
| risk | 风险 | 烟粉 `#C4A7A0` | ⚠️ | 风险预警、需关注 |
| care | 关心 | 雾蓝 `#A0B0C4` | 💙 | 关心提醒、关注对方 |
| promise | 承诺 | 雾绿 `#A0C4A8` | 📌 | 承诺兑现、待执行 |
| followup | 跟进 | 雾金 `#C4C0A0` | 🔄 | 跟进确认、待定 |
| help | 帮助 | 雾紫 `#B0A0C4` | 🤲 | 帮助支持、关系维护 |

### 5.2 各类型详细视觉规范

#### 5.2.1 cooperation_signal（合作信号）— 雾白 #B8C4C0

**标签样式**：
- 背景：`rgba(184, 196, 192, 0.10)`（雾白10%）
- 边框：`1px solid #B8C4C0`（雾白100%）
- 文字：`#B8C4C0` 12px
- 圆角4px，内边距4px 8px

**卡片样式**：
- 左侧色条：`#B8C4C0` 4px
- 图标：🤝 16px，`#B8C4C0`着色
- 匹配分数：若有关联匹配，显示匹配度百分比

**列表中的视觉区分**：
- 色条 + 🤝图标 + "合作信号"标签，三重编码
- 高匹配度合作信号（≥80%）：色条加粗至6px

**详情页视觉**：
- Header背景渐变：`rgba(184, 196, 192, 0.05)` → 透明
- 关联的MatchScoreCard内嵌展示
- AI建议区：雾白左边框3px

#### 5.2.2 risk（风险）— 烟粉 #C4A7A0

**标签样式**：
- 背景：`rgba(196, 167, 160, 0.10)`
- 边框：`1px solid #C4A7A0`
- 文字：`#C4A7A0` 12px

**卡片样式**：
- 左侧色条：`#C4A7A0` 4px

## 6. 资源敏感度UI设计

### 6.1 敏感度定义

| 敏感度 | 值 | 语义 | 匹配行为 |
|--------|-----|------|---------|
| matchable | `matchable` | 可参与匹配 | 正常出现在匹配结果中 |
| no_match | `no_match` | 不可匹配 | 从匹配结果中排除 |

### 6.2 matchable状态（默认）

**视觉表现**：
- PersonCard：正常展示，无额外标识
- 画像页：不显示敏感度区域
- 资源匹配页：正常出现在匹配结果中
- 关系维护页：正常出现在维护提醒中

**设计理由**：matchable是默认状态，绝大多数人属于此状态，显示标识会造成视觉噪音。

### 6.3 no_match状态

**PersonCard中的表现**：
```
┌─────────────────────────────────────┐
│ [头像]  张三                          │
│         阿里巴巴 · AI算法总监           │
│ [AI算法] [CV经验]  🔒不可匹配          │
│                     (灰色半透明遮罩)    │
└─────────────────────────────────────┘
```
- 右上角显示🔒图标 + "不可匹配"文字，`#9CA3AF`
- 卡片整体添加5%灰色遮罩（`rgba(58, 63, 66, 0.05)`）
- 资源标签颜色降饱和度至50%

**画像页中的表现**：
- 敏感度区域显示：🔒 不可匹配 + 原因说明
- 原因文字：12px `#9CA3AF`
- 常见原因：
  - "用户手动标记"
  - "竞争对手，不宜匹配"
  - "敏感关系，需保密"

**资源匹配页中的表现**：
- 不出现在匹配结果列表中
- 底部提示："⚠️ N位联系人标记为不可匹配，已排除"
- 点击提示可查看被排除的人物列表（仅姓名，不展示详情）

**关系维护页中的表现**：
- 仍然出现在维护提醒中（关系维护 ≠ 资源匹配）
- 但卡片上标注🔒标识
- 维护建议不包含"介绍给他人"类建议

### 6.4 敏感度切换交互

**入口**：人物画像页 → 编辑 → 敏感度设置

**切换流程**：
```
点击敏感度设置
    ↓
选择敏感度级别
┌─────────────────────────┐
│  资源匹配敏感度            │
│                          │
│  ○ 可匹配（默认）          │
│  ● 不可匹配               │
│                          │
│  原因（不可匹配时必填）：    │
│  [输入原因_____________]   │
│                          │
│  [取消]        [确认]      │
└─────────────────────────┘
```

**交互规范**：
- 切换为no_match时，必须填写原因（最少5字）
- 切换为matchable时，原因自动清空
- 切换操作需二次确认："确定将张三标记为不可匹配？该联系人将不会出现在资源匹配结果中。"
- 切换回matchable无需二次确认

**安全注意事项**：
- 敏感度变更记录审计日志（谁、何时、从什么改为什么、原因）
- no_match状态的人物数据仍然完整保留，仅影响匹配算法
- 不可批量设置敏感度（防止误操作）

### 6.5 敏感度与callability的区别

| 维度 | sensitivity | callability |
|------|-------------|-------------|
| 类型 | 二值枚举 | 连续值 0.0-1.0 |
| 语义 | 是否允许参与匹配 | 可联系程度 |
| 设置方式 | 用户手动 | 系统计算 + 用户修正 |
| 匹配影响 | 硬排除（不参与） | 软影响（降低权重20%） |
| UI位置 | 画像页编辑 | 画像页展示 |

## 7. 微信小程序WebView适配

### 7.1 认证方案

**Token传递改为ticket模式**：
- 小程序通过 `wx.login()` 获取临时code
- 后端用code换取session_key + openid
- 后端生成一次性ticket（有效期5分钟），通过URL参数传递给WebView
- WebView用ticket换取JWT token
- JWT token存储在sessionStorage中（非localStorage）

**安全注意事项**：
- ticket仅一次使用，换取后立即失效
- JWT token有效期2小时，过期后需重新获取ticket
- 不在URL中传递JWT token（避免泄露风险）
- sessionStorage在WebView关闭时自动清除

### 7.2 H5与小程序通信

```javascript
// H5 → 小程序
wx.miniProgram.postMessage({ data: { action: 'navigate', page: '/person/123' } });
wx.miniProgram.navigateTo({ url: '/pages/person/person?id=123' });

// 小程序 → H5
// 通过URL参数或全局变量传递
```

### 7.3 样式适配

| 项目 | 适配方案 |
|------|---------|
| 安全区域 | `env(safe-area-inset-top/bottom)` |
| 状态栏 | 44px预留空间 |
| 底部导航 | 50px + safe-area-bottom |
| 字体 | 使用系统默认，不强制PingFang SC |
| 滚动 | `-webkit-overflow-scrolling: touch` |
| 点击延迟 | 引入fastclick或使用touch事件 |
| 1px边框 | `transform: scaleY(0.5)` |

### 7.4 存储策略

| 存储类型 | 方案 | 说明 |
|---------|------|------|
| 认证Token | sessionStorage | WebView关闭自动清除 |
| 用户偏好 | sessionStorage | 同上 |
| 临时数据 | 内存变量 | 页面级数据 |
| 持久数据 | 后端API | 不依赖本地持久化 |

**注意**：微信小程序WebView中localStorage可能被清理，且存在隐私风险，统一使用sessionStorage替代。

### 7.5 性能优化

- 首屏HTML < 50KB
- 关键CSS内联
- 图片使用WebP格式 + 懒加载
- 接口请求并发 ≤ 4个
- 避免大列表渲染，使用虚拟滚动
