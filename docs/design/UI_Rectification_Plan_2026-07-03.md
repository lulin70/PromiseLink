## 一、共识决议总览

### 1.1 共识形成方法

由 DevSquad 7 角色（架构师/PM/安全/测试/开发/DevOps/UI 设计师）并行审议 13 项议题，各自从专业维度给出 `采纳 / 调整 / 延后 / 拒绝` 判断。ConsensusEngine 按"产品优先级 PM 一票否决 + 架构与技术方案架构师一票否决 + 安全风险安全工程师一票否决"规则形成最终决议。

## 四、安全加固清单（必须在实施前完成）

> 来源：安全工程师审议报告，按风险等级排序

### 4.1 高风险项（必须在对应功能实施前完成）

| 功能 | 安全基线 | 责任人 |
|------|---------|--------|
| 1.1 隐私数据删除 | DELETE `/privacy/user-data` 前端二次确认（输入"DELETE"短语）+ 后端软删除 30 天保留 + 审计日志 | 后端 + 前端 |
| 1.1 CSV 导入（1.1 批次） | 文件类型白名单（仅 `.csv`）+ MIME 校验 + 大小上限 5MB + 行数上限 + 公式注入过滤（剥离 `=`/`+`/`-`/`@` 开头单元格） | 后端 |
| 1.2 全局搜索 | `query` 长度上限 200 + SQL/NoSQL 注入转义 + `types[]` 枚举白名单 + `date_range` ISO 严格校验 + 多租户隔离（user_id 取自 JWT，非 query） | 后端 |
| 1.2 语义搜索（1.1 批次） | query 进 LLM 前 prompt injection 防护（参考 ai-security-guard 16 模式）+ 系统 prompt 严格分隔 + 响应不返回 embedding 向量 | 后端 |

### 4.2 中风险项

| 功能 | 安全基线 |
|------|---------|
| 1.3 批量操作 | 每个 todo_id 服务端校验归属权；`reminderAction.action` 枚举白名单（complete/snooze/dismiss） |
| 2.1 详情页 | `/entities/:id` 路径参数服务端校验归属权（IDOR 防护） |
| 2.2 action_type | 后端 Pydantic 模型显式定义枚举；过渡期兼容旧 `todo_type` 但 deprecation 日志 |
| 4.3 枚举统一 | 后端枚举迁移脚本 + 存量数据迁移 + 过渡期拒绝 `broken`（422）|

### 4.3 低风险项

| 功能 | 安全基线 |
|------|---------|
| 2.3 引导内容 | 静态文本，禁止 HTML 注入，统一 React 组件渲染 |
| 3.3 SVG 插图 | 本地静态资源；禁用 `<script>`/`onload`/外部引用 |
| 4.1 色彩变量 | 色值不入库，避免存储型 XSS |
