## 四、布局适配问题

| 问题 | 说明 |
|------|------|
| **TabBar与设计稿不一致** | 前端[首页/事件/人脉/待办/承诺] vs 设计稿[首页/人脉/录入/待办/我的]。设计稿把"录入"作为TabBar一级入口(手机版录入页SVG底部高亮✏️录入)，前端用FAB+按钮替代；设计稿有"我的"(设置)，前端无设置页。 |
| **桌面详情栏内容大面积缺失** | 三栏骨架已实现(app.tsx + app.scss ≥1024px sidebar 200px+content flex:1+detail 360px)，但右栏除 events/detail 外全为空状态。与设计稿推进卡详情-桌面版.svg 右栏(需求洞察+话题热区+见面准备+快捷操作)的丰富内容严重不符。 |
| **首页结构未对齐三段式** | 设计稿首页是"问候区+今日日程+紧急待办"三段式+3浮动QuickAction。前端是"统计卡+录入框+导出+事件+待办+供需匹配+健康度+关怀提醒"Dashboard形态，结构与信息层级均不符。 |
| **录入页桌面版缺AI实时预览栏** | 设计稿录入页-桌面版.svg 是三栏(sidebar+输入+AI实时预览)。前端录入页在桌面下仍是单栏流程(提交后轮询)，未实现右侧"AI实时预览"面板。 |
| **推进卡详情桌面版未实现** | 设计稿推进卡详情-桌面版.svg 是三栏丰富布局(左:人物+承诺+交流时间线，右:需求洞察+话题热区+下次见面准备)。前端 `entities/detail.tsx` 是单栏滚动列表，桌面下未做特殊三栏布局。 |
| **新手引导形态错位** | 设计稿首次体验是"记录一次重要交流"4屏交互流(实际录入+确认+获得价值)。前端 `Guide.tsx` 是4步文字说明弹窗(欢迎/三栏布局/记录交流/AI解析)，无实际交互，且内容讲的是"三栏布局"等桌面端功能说明，与设计稿的首次录入引导目标不符。 |

## 五、关键结论

1. **推进卡详情是最大缺口**：设计稿定义12模块+5阶段进度条+桌面三栏，前端仅基础信息+关联列表
2. **首页结构需重构**：从Dashboard统计形态转为三段式(问候+日程+紧急待办)+3浮动QuickAction
3. **TabBar需对齐设计稿**：[首页/人脉/录入/待办/我的]
4. **新手引导形态错位**：设计稿是实际录入交互流，前端是文字说明弹窗
5. **3个P0级完全缺失画面**：ChatPanel(专业版)、首次体验4屏流、(设置页/TTS为P1-P2)
6. **`getDailyReminders` API 已存在但无页面消费**，每日提醒页可较快补齐
