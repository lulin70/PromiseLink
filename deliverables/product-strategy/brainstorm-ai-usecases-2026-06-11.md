## 1. 产品定位与Slogan体系

| 类型 | Slogan | 对应维度 |
|------|--------|---------|
| 主Slogan | **"让每一次连接，都有回应"** | 整体价值承诺 |
| 副Slogan | **"记住需要，兑现承诺，让合作自然发生"** | 三大核心维度 |
| 品牌短句 | **"见面只是开始"** | 情感共鸣 |

**三大维度释义**：
- **记住需要** → 数据输入层 + 智能分析层：AI帮用户记住关系中的一切关键信息
- **兑现承诺** → 行动输出层：AI帮用户追踪和兑现每一句承诺
- **让合作自然发生** → 关系经营层：AI发现隐含关联，让供需自然对接

## 2. 用户研究洞察（来自瑞思）

### 2.1 五大核心痛点

| # | 痛点 | 典型用户原话 | 严重度 |
|---|------|-------------|--------|
| P1 | 承诺遗忘率超80% | "说过的话转头就忘，第二次见面像失忆" | 🔴🔴🔴🔴🔴 |
| P2 | 60-70%联系人静默 | "关系躺在通讯录里死了，不知道怎么活化" | 🔴🔴🔴🔴 |
| P3 | 人脉无法交叉利用 | "帮A找了B，三个月后C也需要B，我完全不记得" | 🔴🔴🔴🔴 |
| P4 | 缺乏系统化提醒 | "关系维护全靠记性，没有提醒机制" | 🔴🔴🔴 |
| P5 | 关系深浅判断模糊 | "该深的关系没深，不该热的关系瞎热" | 🔴🔴🔴 |

### 2.2 用户对AI的期待

| 期待类型 | 最希望AI做的事 | 核心原则 |
|---------|-------------|---------|
| 🔵 替我做 | 信息提取、图谱构建、历史检索、草稿生成 | 低情感、高重复、有明确格式 |
| 🟡 提醒我 | 关键时间节点、承诺兑现、关系冷落、时机建议 | 保留决策权和人际温度 |
| 🔴 别做 | 直接发消息、评价关系、泄露数据、插入广告 | 人际主权+数据主权 |

### 2.3 三种用户态度

| 类型 | 占比 | 核心诉求 | 转化关键 |
|------|------|---------|---------|
| 🟢 拥抱型 | 25-30% | "能自动的都自动，我只想管决策" | 效率提升的可量化证明 |
| 🟡 谨慎型 | 45-55% | "帮我记和提醒，但话要我自己说" | 零学习成本+隐私承诺+融入微信 |
| 🔴 抗拒型 | 15-25% | "我不需要AI，我需要更好的记性" | 不应转化，但叙事上必须区分"AI管理≠AI操控" |

### 2.4 隐私敏感度分级

| 敏感度 | 数据类型 | 愿意度 |
|--------|---------|--------|
| 🟢 低 | 名片、会面时间、职业动态、自己的承诺、关系标签 | 75-90% |
| 🟡 中 | 聊天记录、关系深度评估、语音记录、对方私人信息 | 35-50% |
| 🔴 高 | 关系网络拓扑、商业敏感信息、AI自动发消息、数据训练 | 3-15% |

**核心洞察**：隐私敏感度与"数据是否可被他人利用来复制我的关系优势"直接相关。

### 3.2 五个市场空白点（PromiseLink独占）

1. **承诺追踪（Promise Tracking）** — 所有竞品均为零，PromiseLink已实现双向分析
2. **供需匹配（Supply-Demand Matching）** — 12种关联发现中的supply_demand，竞品无此概念
3. **关系阶段识别** — 7阶段模型（new_connection → long_term_partner），竞品只有"该联系了"
4. **语音关系管理** — NLU/NLG/TTS完整链路，竞品无语音入口
5. **动态优先级预警** — 四维评分+隐式反馈学习，竞品仅有静态排序

### 3.3 最强竞品Clay Earth的三个弱点

1. 搜索功能不可靠（用户直接投诉）
2. 无关系阶段模型（只有"该联系了"没有"关系走到哪一步了"）
3. 正在更名Mesh（过渡期用户可能流失）

## 8. AI功能分层定价建议（来自路径·已按访问方式重构）

### 8.1 为什么按"访问方式"分层，而非"功能额度"

原方案按"200人/50次AI → 解锁更多"分层，本质是在功能上砌墙，用户感知为"你凭什么限制我"。

新方案按"数据在哪里、怎么访问"分层：

| | 基础版 | 专业版 | 定制版 |
|--|--------|--------|--------|
| **定位** | 自己的关系助手 | 随身的关系助手 | 按需定制的关系助手 |
| **部署** | 本地Docker | 本地Docker + 云中继网关 | 云端/私有化部署 |
| **前端** | Taro H5（局域网浏览器） | 微信小程序 | 独立小程序 + Web |
| **AI能力** | 完整（自带Key + 本地模型免费额度） | 完整（含云端AI Key） | 完整 + 定制模型 |
| **访问方式** | 同局域网浏览器 | 随时随地小程序 | 随时随地 |
| **推送通知** | ❌ | ✅ | ✅ |
| **语音录入** | 浏览器录音（基础） | 小程序原生 | 定制 |
| **数据位置** | 家里 | 家里（中继不过夜） | 按需 |
| **安装难度** | Docker一键 | Docker一键 + 扫码连网关 | 我方部署 |
| **定价** | 免费 | ¥98/月 或 ¥888/年 | 定制报价 |
| **核心Slogan** | "数据从不出家门" | "关系助手，随身携带" | "按你的方式来" |

**付费驱动力是真实场景而非人造墙**：基础版用户用着用着，想在外面用手机查看联系人的承诺 → 自然升级专业版。

## 9. 开源策略（已决策 ✅）

### 9.1 决策结论（2026-06-11）

**采用 Open Core 模式（AGPL v3 License）**：
- ✅ 基础版（PromiseLink-Standard）开源
- 🔒 专业版中继网关、微信小程序、定制版全栈闭源
- 📜 License：AGPL v3（防止他人拿开源代码做 SaaS 竞品，用户在隐私场景下信任度更高）

### 9.2 开源范围界定

```
开源（PromiseLink-Standard）：
  ✅ 13步事件处理管线（完整业务逻辑）
  ✅ 实体提取 + 归一算法
  ✅ 所有记忆类型（7种）
  ✅ 6种Todo生成（含Promise双向）
  ✅ 关联发现12种（含供需匹配supply_demand）
  ✅ 完整的本地Docker部署脚本
  ✅ Taro H5 前端（浏览器访问）
  ✅ Obsidian/飞书/企微 适配器
  ✅ 语义搜索、动态优先级评分
  ✅ 关系阶段识别（7阶段模型）

闭源（PromiseLink Pro / 定制版）：
  🔒 中继网关源码（Token控制、推送调度）
  🔒 微信小程序前端（.wxml/.wxss 混淆）
  🔒 语音 NLU/NLG/TTS 云端模型调用封装
  🔒 关系图谱融合算法（定制版专属）
  🔒 云端管理后台（订阅管理、用量仪表盘）
```

### 9.3 为什么开源是基础版的正确决策

| 理由 | 说明 |
|------|------|
| **隐私信任 = 开源证据** | "数据从不出家门"闭源是承诺，开源是可验证的事实。对占45-55%的谨慎型用户是致命一击 |
| **获客杠杆** | 技术用户自己部署、自己验证，写技术博客推荐，把用户变成传播节点 |
| **竞品无人在做** | Clay/Dex/Monica 全闭源 SaaS。开源基础版在"隐私"最敏感维度有结构性优势 |
| **AGPL v3 防竞争** | 云服务提供者修改代码也必须开源。防止别人拿 PromiseLink-Standard 做 SaaS 跟我们竞争 |
| **基础版本来不赚钱** | 它的使命是获客，开源加速这个循环，不损伤营收 |
